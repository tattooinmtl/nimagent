# 2026-06-26

Terminal UI overhaul (animated states, framed input + token panel, bottom status
bar), a full project audit, and the resulting bug fixes + dead-code cleanup.

## Added
- **Multi-state animated status line** in [`src/ui.mjs`](../../src/ui.mjs). One shared
  spinner now drives a registry of colored states, each with its own ANSI color and
  Unicode frames. The original braille `thinking` spinner is preserved.
  - States: `thinking` (magenta), `searching` (cyan), `coding` (green),
    `reading` (blue), `running` (yellow), `tokens` (magenta bar), `timer` (gray clock),
    `ready` (yellow), plus `done`/`error` finalizers.
  - New exports: `startStatus()`, `stopStatus()`, `startGenerationStatus()`,
    `statusDone()`, `hr()`, `promptTop()`, `promptBottom()`, `statusBar()`,
    and `demoStatuses()` (visual demo).
- **Framed token-generation panel** — a yellow-bounded box holding a live token
  bar + elapsed timer, shown while the model produces its first token. Tears down
  the instant text starts streaming (`stopStatus()` is frame-aware).
- **Bottom status bar** — context usage on the left (`used/cap (pct%)`),
  `(provider) model` right-justified to terminal width.
- **Input framing** in [`bin/nimagent.mjs`](../../bin/nimagent.mjs) — the prompt is
  rendered between two gray separator lines, with the status bar above it.
- **Per-action tool animations** in [`src/agent.mjs`](../../src/agent.mjs) — tool calls
  print first, then a single action status (`reading`/`searching`/`coding`/`running`)
  animates while the batch runs in parallel, then results print in call order.

## Fixed
- **🔴 `edit_file` corrupted edits containing `$` patterns**
  ([`src/tools.mjs`](../../src/tools.mjs)). `String.replace(old, new)` interprets
  `$&`, `` $` ``, `$'`, `$$` in the replacement, so inserted code with those tokens
  was silently mangled (e.g. `foo$&bar` → `fooMATCHbar`). Switched to a function
  replacer (`() => new_string`) so replacements are inserted literally.
- **Streaming reported no token usage** ([`src/provider.mjs`](../../src/provider.mjs)).
  Added `stream_options: { include_usage: true }` so spec-compliant providers
  (NVIDIA NIM) emit a final usage chunk. Without it `/cost` and the new status bar
  always showed 0 tokens.
- **`--resume` CLI flag was dead** ([`bin/nimagent.mjs`](../../bin/nimagent.mjs)).
  It was parsed but never used. Wired it to rebuild the conversation at startup via
  a shared `restoreSessionMessages()` helper (also used by the `/resume` command).
- **Restored token-by-token streaming** in [`src/agent.mjs`](../../src/agent.mjs)
  after the UI work had buffered it, and fixed a missing `streamWrite`/`streamNewline`
  import that would have thrown `ReferenceError` at runtime.

## Changed
- `argSummary()` ([`src/agent.mjs`](../../src/agent.mjs)) now shows useful detail for
  `run_test`, `jq_query`, `web_search`, `move_file`/`copy_file` (`from → to`), and
  `delete_path`/`make_dir`.
- Idle input-frame separators are gray; yellow is reserved for the active
  token-generation panel and alerts.

## Removed (dead code)
- [`src/provider.mjs`](../../src/provider.mjs): the no-op `node:http`/`node:https`
  `Agent` keep-alive code + `getAgent()` (ignored by undici's global `fetch`), and
  the unused non-streaming `chat()` export.
- [`src/config.mjs`](../../src/config.mjs): never-called `Session` methods
  `estimateTokens`, `addMessageTokens`, `getCompactionSuggestion`, `shouldCompact`,
  and the unused `_messageTokens` field.
